# Local Banner Bot

Локальный Telegram-бот для генерации баннеров по документам без n8n.

## Что уже умеет

- принимает ссылку на Google Doc, публичную ссылку на Word/OneDrive документ или `.docx` файлом в Telegram
- читает Google Docs через Google Docs API и Word/docx через прямую загрузку
- вытаскивает H1 и H2
- после документа принимает одну референс-картинку и использует ее как источник стиля для всей серии
- генерирует визуальную основу через Kie
- собирает финальный баннер локально через `sharp`
- показывает баннеры в Telegram
- поддерживает точечный `retry` по кнопке `Переделать`
- при retry принимает текстовые правки и перегенерирует текущий баннер

## Подготовка

1. Установи Node.js 20+ (рекомендуется Node 22; см. `.nvmrc`)
2. Скопируй `.env.example` в `.env`
3. Заполни переменные окружения
4. Если используешь Google Doc, убедись, что он расшарен на сервисный аккаунт с правом чтения
5. Установи зависимости:

```bash
npm install
```

## Запуск

```bash
npm start
```

## Команды бота

- `/start` — начать новую сессию
- `/reset` — сбросить текущую сессию

## Поток работы

1. Отправляешь `/start`
2. Отправляешь ссылку на документ или `.docx` файлом в Telegram
3. Бот разбирает документ и просит одну референс-картинку стиля
4. Отправляешь референс-картинку или пропускаешь этот шаг
5. Бот генерирует серию баннеров в стиле референса
6. Если конкретный баннер нужно поправить, нажимаешь `Переделать`
7. Отправляешь текст правок
8. Бот перегенерирует только выбранный баннер

## Ограничения текущей версии

1. Состояние хранится в памяти процесса
2. Если процесс перезапустить, текущие сессии сбросятся
3. Типографика пока на системном жирном шрифте
4. Для идеального попадания в бренд можно позже добавить логотип и фирменный `.ttf`

## Linux Server Quick Start

```bash
git clone https://github.com/SamKlvr/banner-bot.git
cd banner-bot
cp .env.example .env
# заполнить .env своими токенами
# обязательно для Linux-сервера:
npm install
npm start
```

Важно: на Linux-сервере перед первым запуском обязательно выполнить `npm install`, потому что зависимости в репозитории не хранятся.

Если сервер запускается через `systemd` или `pm2`, сначала проверь обычный ручной старт через `npm start`.
